    Returns a Buffer Array of the raw Buffers of the EIP-2930 transaction, in order.

    Format: [chainId, nonce, gasPrice, gasLimit, to, value, data, accessList, signatureYParity (v), signatureR (r), signatureS (s)]

    Use AccessListEIP2930Transaction.serialize to add a transaction to a block with {@link Block.fromValuesArray}.

    For an unsigned tx this method uses the empty Buffer values for the signature parameters v, r and s for encoding. For an EIP-155 compliant representation for external signing use AccessListEIP2930Transaction.getMessageToSign.

    Returns AccessListEIP2930ValuesArray

supports

  • supports(capability: Capability): boolean

  • Inherited from ReadOnlyValidEIP2930Transaction.supports

    • Defined in Users/sbacha/hardhat/node_modules/@ethereumjs/tx/dist/baseTransaction.d.ts:83

    Checks if a tx type defining capability is active on a tx, for example the EIP-1559 fee market mechanism or the EIP-2930 access list feature.

    Note that this is different from the tx type itself, so EIP-2930 access lists can very well be active on an EIP-1559 tx for example.

    This method can be useful for feature checks if the tx type is unknown (e.g. when instantiated with the tx factory).

    See Capabilites in the types module for a reference on all supported capabilities.

    Parameters

    • capability: Capability

    Returns boolean
